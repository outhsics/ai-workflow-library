# 一条内容工作流：从选题到发布

## 输入

- 一个方向明确的问题
- 若干参考链接或资料
- 目标平台和输出形式

## 处理链路

1. 用搜索型模型确认资料面。
2. 用写作型模型整理结构和提纲。
3. 用编码代理或脚本处理格式、配图、站点发布。
4. 用知识库保存最终模板。

## 工具搭配示例

| 阶段 | 工具 | 用途 |
| --- | --- | --- |
| 选题 | Perplexity / Gemini | 补全外部信息 |
| 整理 | ChatGPT / Claude | 提炼结构 |
| 落地 | Cursor / Codex / Ghostty | 写页面、调样式、跑构建 |
| 发布 | GitHub Actions | 自动部署 |
| 沉淀 | Obsidian | 存模板与复盘 |

## 最容易出问题的地方

- 资料没有先归拢，导致写到一半又去补搜索
- 模型与执行工具职责不清，反复切换
- 发布过程依赖手工步骤，无法稳定复用

## 可以继续扩展的方向

- 增加图像生成或封面工作流
- 增加多平台分发流程
- 增加数据回收和复盘页
