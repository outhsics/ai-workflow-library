# 告别 `cd`：终端神器 `zoxide` 使用指南

如果你每天都在终端里切项目，`cd` 最大的问题不是不会用，而是太机械。

进入一个深层目录，经常要么手打一长串路径，要么疯狂按 `Tab`。  
`zoxide` 的价值在于，它把“记路径”这件事从你脑子里拿走了。

## 它到底做了什么

`zoxide` 会记录你去过的目录，并根据：

- 访问频率
- 最近访问时间

给目录打分。之后你不再需要完整路径，只需要输入几个关键词，它就能猜到你要去哪里。

## 最常用的三个命令

### 1. 直接跳目录

```bash
z project
```

如果你平时经常进某个项目目录，记住一个关键词就够了。

### 2. 组合关键词精准定位

```bash
z admin src
```

当目录重名很多时，用多个词一起筛。

### 3. 交互式模糊跳转

```bash
zi
```

如果你装了 `fzf`，这一步会非常顺：输入片段，选中结果，回车直接进入。

## 为什么它比单纯路径补全更好

路径补全还是要求你记得“结构”。  
`zoxide` 要求你只记得“印象”。

这在项目多、配置目录多、经常跨仓库切换时差别很明显。

## 新装后最容易遇到的问题

### 为什么提示找不到目录

因为它还没学会你的习惯。

解决方式很简单：

- 先正常 `cd` 进去几次
- 或者手动 `zoxide add <path>`

一旦它记住，后面就会越来越准。

## 适合谁

- 仓库多的人
- 经常在终端切目录的人
- 讨厌手敲长路径的人

## 我会怎么把它接进工作流

如果我记得项目名：

```bash
z 项目名
```

如果我只记得大概位置但不记得名字：

```bash
yy
```

也就是说：

- `zoxide` 负责“直接瞬移”
- `Yazi` 负责“边看边找”

这两个并不冲突，反而是互补关系。

## 一句话总结

从 `cd` 切到 `zoxide`，本质上不是换了一个命令，而是把“目录导航”从手工输入，变成了基于习惯的快速跳转。
