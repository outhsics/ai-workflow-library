# Obsidian 完全免费同步方案：Git + GitHub

如果你不想为 Obsidian 官方同步付费，又希望保留版本历史、多设备可迁移和长期可控性，`Git + GitHub` 是一条很扎实的路线。

## 为什么这条方案值得选

- 免费
- 设备数量不受限制
- 自带版本控制
- 迁移成本低
- 对开发者友好

它不是最无感的方案，但很适合愿意自己掌控数据的人。

## 这套方案的核心结构

你可以把它理解成三层：

1. 本地 Obsidian Vault
2. Git 仓库负责版本记录
3. GitHub 仓库负责多设备同步

电脑上通过 Git 提交和推送。  
其他设备通过拉取最新内容保持一致。

## 最适合谁

- 已经有 Git 基础
- 希望笔记有完整历史
- 不介意偶尔手动处理同步
- 希望数据完全在自己可控范围内

## 最小搭建步骤

### 1. 初始化仓库

在你的 Vault 目录中：

```bash
git init
git add .
git commit -m "init obsidian vault"
```

### 2. 创建远端仓库

推荐使用私有仓库同步个人笔记。

```bash
git remote add origin git@github.com:<your-name>/<your-vault>.git
git branch -M main
git push -u origin main
```

### 3. 养成同步动作

电脑端最简单的方式是固定成两步：

```bash
git pull origin main
git add -A && git commit -m "sync notes" && git push origin main
```

如果你不想每次都手敲，可以封装成别名或脚本。

## 手机端怎么做

如果你需要在手机上同步，有两种方向：

### 方案 A：Obsidian Git 插件

适合希望尽量图形化的人。

### 方案 B：Termux + Git

适合更熟悉命令行的人。

两种路线本质都一样：  
让手机端也具备拉取、提交、推送的能力。

## 这条方案最容易出的问题

### 1. 冲突

多个设备同时修改同一篇笔记时，冲突是最常见的问题。

避免冲突的关键不是技巧，而是习惯：

- 编辑前先拉最新
- 一台设备改完就尽快推送
- 不要长时间多端并行编辑同一篇笔记

### 2. 仓库变大

如果 Vault 里有很多图片、PDF、压缩包，仓库会越来越重。

应对方式：

- 用 `.gitignore` 排除不必要的大文件
- 定期清理历史和废弃附件
- 让附件目录保持可控

### 3. 认证失败

如果你走 HTTPS，通常需要 token；如果走 SSH，需要把密钥配置好。

## 为什么我仍然推荐它

因为这条路线不只是“同步”，它还顺手解决了三件事：

- 备份
- 历史回溯
- 跨设备迁移

这三个价值叠在一起，就让它不再只是一个省钱方案，而是一个更长期的知识库基础设施方案。

## 一句话总结

如果你想要的是“最傻瓜”的体验，这不是第一选择。  
如果你想要的是“免费、可控、可回滚、可迁移”，那 `Git + GitHub` 依然是 Obsidian 很强的一条路。
