# Yazi 入门与我的配置

Yazi 是我在终端里最常用的文件管理器。它替代的不是 Finder 本身，而是“为了找一个目录而不断 `cd`、`ls`、补全路径”的那部分摩擦。

## 为什么我会长期保留它

- 比纯命令行找目录更直观
- 比图形文件管理器更快回到终端执行
- 能把“浏览目录”直接接到“继续编码”上

真正让我留下来的，不是它功能多，而是它和终端工作流接得很顺。

## 我最常用的启动方式

```bash
yy
```

`yy` 和普通的 `y` 最大区别是：

- `y`：只是打开 Yazi
- `yy`：退出后终端会自动切到你最后浏览的目录

这一个差异，决定了它是不是“开发工作流工具”。

## 新手先记住这 10 个键就够了

- `j / k`：上下移动
- `h / l`：返回上级 / 进入目录
- `Enter`：打开文件或进入目录
- `q`：退出
- `Space`：选中
- `y / x / p`：复制 / 剪切 / 粘贴
- `.`：显示或隐藏隐藏文件
- `s / S`：按文件名 / 按内容搜索
- `~`：帮助

如果这些都熟了，80% 的日常场景都已经能覆盖。

## 我这类用户最适合的三个场景

### 1. 快速进入项目目录

```bash
yy
```

找完目录退出，终端已经切进去，接着可以直接：

```bash
claude
```

或者：

```bash
lg
```

### 2. 整理下载目录

Yazi 很适合做轻量文件整理：

- `Space` 多选
- `x` 剪切
- 切到目标目录
- `p` 粘贴

### 3. 浏览配置目录

比如看 `~/.config`、`.ssh`、`.git` 这类隐藏目录时，Yazi 比手敲路径舒服很多。

## 我偏好的配置方向

我更倾向于“低复杂度版”的 Yazi，而不是堆很多插件：

- 自然排序
- 目录优先
- 合理的三栏比例
- 够用的预览能力
- 主题跟随系统浅深色

这样做的好处是：

- 稳定
- 维护成本低
- 换机器时容易迁移

## 一条最顺的配合方式

我的典型顺序是：

1. 打开终端
2. `yy`
3. 找到项目目录
4. 退出
5. `claude`
6. 改完以后 `lg`

所以 Yazi 在这里不是独立工具，而是整个终端工作流的第二步。

## 一句话总结

如果你经常在终端里工作，Yazi 最值得用的点不是“文件管理器”这四个字，而是它能把“找目录”这件事变得足够轻，轻到你愿意天天用。
