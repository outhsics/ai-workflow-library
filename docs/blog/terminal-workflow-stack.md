# 终端工作流速查表：Ghostty + Yazi + Claude + Lazygit

如果只保留一条最顺手的终端路径，我会选这套：

```bash
ghostty
yy
claude
lg
```

它的重点不是“工具多”，而是每个工具都只负责一件事。

## 这四个工具分别负责什么

- `Ghostty`：终端入口，承载后续所有动作
- `Yazi`：找目录、浏览文件、快速进入项目
- `Claude`：读代码、改代码、生成说明和脚本
- `Lazygit`：看改动、拆提交、提交推送

这条链路很适合“我要尽快开始干活，而不是先找半天目录”的场景。

## 最简工作流

### 1. 打开终端

```bash
ghostty
```

### 2. 找到项目目录

```bash
yy
```

`yy` 的价值在于：退出 Yazi 之后，终端会自动切到你最后浏览的目录。

### 3. 进入 AI 编程

```bash
claude
```

如果你在做终端代理式编码，这一步会比“先手动全读项目”快很多。

### 4. 改完后看 Git

```bash
lg
```

这时最适合做三件事：

- 看哪些文件变了
- 拆分提交粒度
- 写清楚 commit message

## 这套组合为什么顺

### Ghostty 负责承载

你需要一个稳定、响应快、适合分屏的终端。  
如果终端本体不顺，后面的工作流也会打折。

### Yazi 负责替代“找路径”

很多时间不是花在写代码，而是花在“项目到底在哪个目录”。  
`yy` 这一步把找目录和切目录合并了。

### Claude 负责“进入执行状态”

不是把它当聊天工具，而是把它当“读仓库和落地修改”的入口。

### Lazygit 负责“收尾”

人最容易在最后一步偷懒，结果改动一大坨。  
`lg` 让提交这件事更轻，所以更容易形成小步提交习惯。

## 适合的三个高频场景

### 场景 1：打开项目开始编码

```bash
ghostty
yy
claude
```

### 场景 2：改完代码检查改动

```bash
lg
```

### 场景 3：临时找配置文件

```bash
yy ~/.config
```

## 这条工作流背后的原则

不是去追求“最强工具链”，而是让动作尽量少切换：

- 找目录用一个工具
- 编码用一个工具
- Git 收尾用一个工具

一条顺的日常路径，往往比单个工具多几个功能更重要。
